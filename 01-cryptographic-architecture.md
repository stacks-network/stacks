# Cryptographic Architecture: The Shift to 24-Word Mnemonics

## 1. Executive Summary
The Stacks ecosystem has undergone a fundamental realignment regarding developer identity and transaction authorization. The Clarinet CLI, the industry standard for Clarity smart contract development, has deprecated raw private keys and 12-word mnemonics in favor of mandatory **24-word BIP-39 mnemonic configurations**.

This document analyzes the cryptographic primitives driving this decision, exploring the shift from ad-hoc secrets to institutional-grade Hierarchical Deterministic (HD) security.

## 2. The Limits of Raw Private Keys
Historically, early Stacks tooling allowed the use of a `secret_key`—a raw 256-bit integer represented as a 64-character hex string. While simple, this approach is architecturally brittle.

### 2.1 The ECDSA Constraint
Stacks utilizes the **Elliptic Curve Digital Signature Algorithm (ECDSA)** on the secp256k1 curve.
* **Private Key:** A random integer selected from the curve order $n$.
* **Limitation:** A raw private key corresponds to exactly one public key. It is a "singular" secret with no inherent hierarchy.



### 2.2 The "Simulation" Problem
Clarinet is not just a deployer; it is a simulation engine (`Simnet`). To accurately simulate a blockchain environment, the runtime must instantiate multiple actors (Deployer, Wallet 1, Wallet 2, etc.) to test permissions and transfers.

* **Raw Key Failure:** A raw private key cannot mathematically generate "child" wallets. It is an endpoint.
* **Mnemonic Success:** A mnemonic serves as a "root seed." From this root, Clarinet uses **BIP-32** to deterministically derive an infinite tree of valid, funded accounts. This capability is essential for the `Clarinet console` and `Clarinet integrate` features.

## 3. BIP-39: The Mathematics of Entropy
The core friction point for many developers is the rejection of 12-word mnemonics in Clarinet v2.15.0+. This is a strict enforcement of entropy standards.

### 3.1 12 Words vs. 24 Words
The length of the mnemonic directly correlates to the underlying binary entropy generated by the wallet.

| Feature | 12-Word Mnemonic | 24-Word Mnemonic |
| :--- | :--- | :--- |
| **Entropy Bits** | 128 bits | 256 bits |
| **Checksum** | 4 bits | 8 bits |
| **Total Bits** | 132 bits | 264 bits |
| **Security** | $2^{128}$ combinations | $2^{256}$ combinations |

**The Clarinet Mandate:** Clarinet enforces 24 words to ensure **256-bit security**, aligning Stacks development with the security profiles of hardware wallets like Ledger and Trezor.



## 4. Derivation Paths (BIP-44)
Once the seed is restored, the "map" to the funds is provided by the derivation path. Stacks utilizes a specific registry under BIP-44.

**Standard Stacks Path:** `m / 44' / 5757' / 0' / 0 / 0`

* `44'`: Purpose (BIP-44 Standard)
* `5757'`: Coin Type (Stacks/STX)
* `0'`: Account Index
* `0`: Change (External Chain)
* `0`: Address Index

**Critical Configuration Note:** If a mnemonic is valid (24 words) but the wallet appears empty, the issue is almost always a derivation path mismatch. Wallets created in Bitcoin-centric tools may default to Coin Type `0'`, rendering the Stacks address invisible to Clarinet.

## 5. Architectural Conclusion
The transition to 24-word mnemonics is not merely a syntax change; it is an architectural requirement for multi-actor simulation and a security upgrade to 256-bit entropy. The `bip39 error` seen in the CLI is a deliberate guardrail enforcing these standards.
